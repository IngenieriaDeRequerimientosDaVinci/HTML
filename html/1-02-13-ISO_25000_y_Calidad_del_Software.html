ISO 25000 - Calidad de Software (SQuaRE)

Características Principales

Funcionalidad

Capacidad del software para proporcionar funciones que satisfacen las necesidades declaradas e implícitas cuando se usa bajo condiciones específicas.

Confiabilidad

Capacidad del sistema para mantener un nivel específico de funcionamiento cuando se usa bajo condiciones y período de tiempo determinados.

Usabilidad

Capacidad del producto software para ser entendido, aprendido, usado y resultar atractivo para el usuario cuando se usa bajo condiciones específicas.

Eficiencia

Desempeño relativo a la cantidad de recursos utilizados bajo determinadas condiciones, incluyendo tiempo de respuesta y uso de recursos.

Mantenibilidad

Capacidad del producto software para ser modificado efectiva y eficientemente, debido a necesidades evolutivas, correctivas o perfectivas.

Portabilidad

Capacidad del producto software para ser transferido de forma efectiva y eficiente de un entorno hardware o software a otro.

Seguridad

Capacidad de protección de la información y los datos de manera que personas o sistemas no autorizados no puedan leerlos o modificarlos.

Compatibilidad

Capacidad de dos o más sistemas o componentes para intercambiar información y/o llevar a cabo sus funciones requeridas cuando comparten el mismo entorno.

Implementación en Proyectos

Característica Beneficio
Evaluación sistemática Mejora continua del proceso de desarrollo mediante la identificación temprana de problemas y oportunidades de mejora.
Métricas establecidas Medición objetiva de la calidad que permite tomar decisiones basadas en datos y establecer objetivos cuantificables.
Marco estandarizado Consistencia en la evaluación que facilita la comparación entre diferentes productos y versiones del software.
